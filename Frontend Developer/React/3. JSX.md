
# Question 2: What is JSX?

### Mid-Level Answer

> JSX (JavaScript XML) is a syntax extension for JavaScript that allows us to write HTML-like code inside JavaScript. It makes React components more readable and easier to write because the UI and the component logic can be kept together.
> 
> Although JSX looks like HTML, it is **not HTML**. Browsers cannot understand JSX directly. During the build process, tools like **Babel** transpile JSX into regular JavaScript using functions such as `React.createElement()` (or the automatic JSX runtime in newer React versions).
> 
> React then uses the resulting JavaScript objects to create the Virtual DOM.

---

### Example

JSX:

```jsx
function App() {
  return <h1>Hello, Mukles!</h1>;
}
```

Babel converts it (conceptually) to:

```javascript
function App() {
  return React.createElement("h1", null, "Hello, Mukles!");
}
```

Or, with the modern JSX transform, it compiles to calls from the `react/jsx-runtime` package rather than requiring `React.createElement` directly.

---

### Why do we use JSX?

- Easier to read and write than nested `React.createElement()` calls.
    
- Allows JavaScript expressions inside UI using `{}`.
    
- Improves maintainability.
    
- Enables React to describe the UI declaratively.
    

---

### Example of JavaScript inside JSX

```jsx
const name = "Mukles";

function App() {
  return <h1>Hello, {name}</h1>;
}
```

The expression inside `{}` is evaluated by JavaScript before rendering.

## Follow-up Questions (Very Common)

After JSX, the interviewer may ask:

### 1. Why do we use `className` instead of `class`?

Expected answer:

- `class` is a reserved keyword in JavaScript.
    
- JSX uses `className` to set the HTML `class` attribute.
    

---

### 2. Can we write JavaScript inside JSX?

Expected answer:

- Yes, using curly braces `{}`.
    
- You can use expressions such as variables, function calls, arithmetic, ternary operators, and array methods like `map`.
    

Example:

```jsx
const age = 24;

return <h1>{age >= 18 ? "Adult" : "Minor"}</h1>;
```

---

### 3. Can we use `if` statements inside JSX?

Expected answer:

- Not directly inside JSX because JSX expects expressions, not statements.
    
- Use a ternary operator, logical `&&`, or perform the `if` check before the `return`.

### 🎯 Interview Tip

A mid-level candidate should mention **Babel** and explain that **JSX is syntax that gets transpiled into JavaScript**. Saying only _"JSX lets us write HTML in JavaScript"_ is incomplete because JSX is not actually HTML, and understanding that distinction shows a stronger grasp of how React works internally.
