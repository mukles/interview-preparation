
# Question 3: What are Props and State? What's the difference between them?

This question almost always comes after React and JSX.

---

# Mid-Level Answer

> **Props** (short for _properties_) are used to pass data from a parent component to a child component. They are **read-only**, meaning a child component should not modify the props it receives. Props make components reusable because the same component can display different data depending on the values passed by the parent.
> 
> **State** is data managed within a component. Unlike props, state is **mutable** through React's state update functions such as `setState` in class components or the setter returned by `useState` in functional components. When state changes, React re-renders the component so the UI reflects the new state.

---

## Example of Props

```jsx
function User({ name }) {
  return <h1>Hello, {name}</h1>;
}

function App() {
  return <User name="Mukles" />;
}
```

Here:

- `App` is the parent.
    
- `User` is the child.
    
- `"Mukles"` is passed as a prop.
    

The child **cannot modify** `name`.

---

## Example of State

```jsx
function Counter() {
  const [count, setCount] = useState(0);

  return (
    <>
      <h1>{count}</h1>
      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>
    </>
  );
}
```

Here:

`count` belongs to the component.

When

```jsx
setCount(count + 1)
```

is called,

React updates the state and re-renders the component.

---

# Props vs State

|Props|State|
|---|---|
|Passed from parent|Managed inside the component|
|Read-only|Can be updated via state setters|
|Used for communication|Used for component data|
|Parent controls it|Component controls it|
|Changes when parent passes new values|Changes when the component updates it|

# Common Follow-up Questions

After this answer, interviewers often ask:

### 1. What happens when props change?

Expected answer:

> When a parent component passes new props, React re-renders the child component. During reconciliation, React compares the new Virtual DOM with the previous one and updates the Real DOM only if necessary.

---

### 2. Can a child component modify props?

Expected answer:

> No. Props are immutable from the child component's perspective. If data needs to change, the child can call a callback function provided by the parent, and the parent updates its state.

---

### 3. When should you use state instead of props?

Expected answer:

> Use state for data owned and managed by the component. Use props to pass data or behavior from a parent to a child.

---

### 4. If the parent re-renders, does the child always re-render?

This is a favorite mid-level question.

The expected answer is:

> By default, yes. When a parent re-renders, React re-renders its child components. However, if the child is wrapped with `React.memo` and its props haven't changed, React can skip re-rendering that child. Similarly, hooks like `useMemo` and `useCallback` can help avoid unnecessary recalculations or unstable function references.
