

# Interview Answer

> The **Virtual DOM** is a lightweight JavaScript representation of the Real DOM. It's not the actual DOM that the browser renders—it's an in-memory copy that React uses to determine what has changed in the UI.
> 
> Whenever a component's state or props change, React creates a new Virtual DOM tree and compares it with the previous one through a process called **reconciliation**. Based on this comparison, React identifies the minimum set of changes required and updates only those parts of the Real DOM instead of rebuilding the entire page.
> 
> This approach reduces unnecessary DOM manipulations, which are relatively expensive operations, and helps improve rendering performance, especially in large applications.

---

# Example

Suppose we have:

```jsx
function App() {
  const [count, setCount] = useState(0);

  return <h1>{count}</h1>;
}
```

Initially:

```text
Virtual DOM
------------
<h1>0</h1>

Real DOM
---------
<h1>0</h1>
```

After:

```jsx
setCount(1);
```

React creates a **new Virtual DOM**.

```text
Old Virtual DOM
---------------
<h1>0</h1>

New Virtual DOM
---------------
<h1>1</h1>
```

React compares them.

It notices that **only the text changed**.

Instead of rebuilding the entire page, React updates only:

```html
<h1>1</h1>
```

---

# Why not update the Real DOM directly?

Because DOM operations are relatively expensive.

Updating the Real DOM may involve:

- Layout (Reflow)
    
- Paint (Repaint)
    
- Composite
    

If React updated the Real DOM on every state change, performance would suffer in large applications.

Instead, React:

```
State changes
      ↓
Create new Virtual DOM
      ↓
Compare with old Virtual DOM
      ↓
Find differences (Diffing)
      ↓
Update only changed nodes
      ↓
Real DOM
```

---

# Does React always update the Real DOM?

No.

If React compares the old and new Virtual DOM and finds no meaningful difference, it doesn't make unnecessary updates to the Real DOM.

---

# Is the Virtual DOM why React is fast?

Not by itself.

React's performance comes from a combination of:

- Efficient reconciliation (diffing)
    
- Batched state updates
    
- Selective DOM updates
    
- React Fiber scheduling
    
- Features like `React.memo`, `useMemo`, and `useCallback` when used appropriately
    

The Virtual DOM is one part of that overall strategy.
