# Performance Optimization Roadmap

You should know **when to use each technique**, not just what it does.

## 1. Why does React re-render?

Interview question:

> **When does a React component re-render?**

Expected answer:

A component re-renders when:

- State changes
    
- Props change
    
- Context changes
    
- Parent re-renders (by default)

## 3. useMemo ⭐⭐⭐⭐⭐

Question:

> When should you use useMemo?

Not:

> It improves performance.

Instead:

> It memoizes the result of an expensive computation so React can reuse it when dependencies haven't changed.

Example:

```jsx
const sortedUsers = useMemo(() => {
  return users.sort(compareUsers);
}, [users]);
```

---

## 4. useCallback ⭐⭐⭐⭐⭐

Question:

> Why use useCallback?

Answer:

It memoizes a function reference to avoid creating a new function on every render, which is particularly useful when passing callbacks to memoized child components.

---

## 5. React.memo + useCallback ⭐⭐⭐⭐⭐

A favorite interview question:

> Why isn't React.memo enough?

Because if you pass a newly created function every render:

```jsx
<Child onClick={() => doSomething()} />
```

the function reference changes each time, so `React.memo` sees the props as different and re-renders the child.

Use:

```jsx
const handleClick = useCallback(() => {
  doSomething();
}, []);

<Child onClick={handleClick} />
```

# Real-world Questions

Instead of asking:

> What is useMemo?

they'll ask:

> Your table has 10,000 rows and typing in the search input feels slow. How would you optimize it?

Or:

> Your child component renders 200 times. How do you debug and fix it?

Or:

> The API is called on every keystroke. What would you do?

Or:

> Why is React.memo not working?

These scenario-based questions are common because they reveal whether you've solved real performance problems.
